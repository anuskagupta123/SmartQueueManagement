# Deploy On A Free Server (Render)

This project can be deployed on Render free tier using the blueprint in [render.yaml](render.yaml).

## 1. Push your code

Push this repository to GitHub (or GitLab).

## 2. Create the Render service

1. Open Render dashboard.
2. Click New + and choose Blueprint.
3. Connect your repo.
4. Render will read [render.yaml](render.yaml) and create queue-optimizer-api.

## 3. Set required environment variables

In Render service settings, set:

1. DATABASE_URL: your Postgres connection string.
2. SESSION_SECRET: already auto-generated by render.yaml, rotate if needed.
3. RESEND_API_KEY: optional, only needed for real email sending.

Notes:

1. PORT is provided by Render automatically.
2. Health check endpoint is /api/healthz.

## 4. Optional frontend deployment

You can deploy artifacts/smartqueue as a static site on any free host (Render Static Site, Netlify, Vercel).

Set this environment variable on the frontend host:

1. VITE_API_BASE_URL: your Render API URL, for example https://queue-optimizer-api.onrender.com

The frontend now supports this variable via [artifacts/smartqueue/src/main.tsx](artifacts/smartqueue/src/main.tsx).
